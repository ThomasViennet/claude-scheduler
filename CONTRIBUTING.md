# Contributing to claude-scheduler

Thanks for your interest in contributing! This project welcomes issues and pull requests.

## Reporting issues

If you find a bug or want to request a feature, please open an issue. Before doing so:

- Search existing issues to avoid duplicates.
- Use the provided issue templates when possible.
- Include clear steps to reproduce for bugs, and your environment details.

## Submitting a pull request

1. Fork the repository and create your branch from `main`.
2. Make your changes with clear, focused commits.
3. Test your changes locally before submitting.
4. Open a pull request describing what you changed and why.

All pull requests are reviewed before being merged. Please be patient while your PR is reviewed, and respond to any requested changes.

## Guidelines

- Keep pull requests focused on a single change when possible.
- Follow the existing code style of the project.
- Update documentation if your change affects usage.

## Questions

If you have a question, feel free to open an issue with the "question" label.
